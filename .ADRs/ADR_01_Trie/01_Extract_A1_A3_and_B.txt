WHY THIS PHASE EXISTS (read before coding)
The DNSBL matching logic lives inline inside the ~450-line `operate()` function
in src/usr/local/pkg/pfblockerng/pfb_unbound.py, tangled with qstate mutation
and I/O. Before any data structure can be swapped, the matching must be pulled
into small pure functions that can be pinned by tests and later reimplemented.
This phase extracts the remaining inline walk (HSTS), deduplicates the
feed/group lookup, introduces one canonical suffix-iteration generator, and
makes all matcher signatures consistent so that trie versions become drop-in
replacements in later phases.

ACTION PLAN (ordered)
1. A3 first — `iter_domain_suffixes`.
   Add a pure generator:
       def iter_domain_suffixes(name: str) -> Iterator[str]
   It must yield the same sequence of progressively-shorter suffixes that the
   existing `q = q.split('.', 1)[-1]` loops produce, STARTING FROM `name`
   itself (i.e. yields name, then name with leftmost label removed, ... down to
   the last label). Do NOT yet change any matcher to use it in a way that alters
   output — first prove equivalence with a quick local check, then refactor
   `find_zone_match`, `find_noaaaa_wildcard_parent`, and `whitelist_check_domain`
   to use it INTERNALLY only where the iteration is identical. IMPORTANT: the
   three existing walkers do NOT share the same start offset or stride:
     - find_zone_match: starts at full name, stride 1, includes self.
     - find_noaaaa_wildcard_parent: starts at PARENT (name.split('.',1)[-1]), stride 1.
     - whitelist_check_domain: exact + 'www.'-strip + parent walk gated by `x >= tld_seg`.
   Preserve each walker's exact start/stride/gate. If a walker's iteration does
   not map cleanly onto the generator, leave that walker's loop as-is rather
   than forcing it. The generator is a shared primitive, not a behavior change.

2. A2 — `resolve_feed_group`.
   Add:
       def resolve_feed_group(index: Any) -> tuple[Any, Any]
   returning (feed, group) with the SAME fallback behavior as the inline code
   (when `feedGroupIndexDB.get(index)` is None, the caller currently leaves the
   pre-initialized defaults `feed = 'Unknown'`, `group = 'Unknown'`). Decide and
   document: either return ('Unknown','Unknown') on miss, OR return a sentinel
   the caller maps to its existing defaults. Match current observable behavior
   exactly. Replace BOTH inline occurrences (data branch, zone branch) in
   operate() with calls to this function.

3. A1 — `hsts_check_domain`.
   Extract the inline HSTS block into:
       def hsts_check_domain(name: str, hsts_db: dict[str, Any],
                             hsts_tlds: tuple[str, ...] | list[str], tld: str)
                             -> tuple[bool, str]
   returning (is_in_hsts, p_type) where p_type is 'HSTS_TLD' when the TLD is in
   hsts_tlds, 'HSTS' when a suffix membership hit occurs, and unchanged
   otherwise. CRITICAL: preserve the step −2 stride
   (`range(q.count('.') + 1, 0, -2)`). Do not silently change it to −1. In
   operate(), replace the inline block with a call; keep the surrounding
   nullBlocking / p_type assignment behavior identical.

4. B — Unify signatures.
   Normalize the matcher helpers to a consistent leading argument order:
   `(container, name, *opts)`. Today find_zone_match is `(q_name, zone_db)` while
   whitelist_check_domain is `(name, white_db, tld_seg)`. Pick `(container, name, ...)`
   (container first) OR `(name, container, ...)` (name first) — choose whichever
   requires the fewest call-site edits, apply it to ALL domain matchers
   (find_zone_match, find_noaaaa_wildcard_parent, whitelist_check_domain,
   hsts_check_domain), and update every call site in operate(). Keep return
   shapes unchanged.

5. Run the suite and linters (see VERIFICATION).
