WHY THIS PHASE EXISTS
There is no live Unbound in CI; correctness of the trie swap can only be
guaranteed by comparing the new implementation against the current one. After
Phases 1-2 the matchers and orchestration are pure functions with stable
signatures — ideal oracle targets. Several existing tests also poke module
globals directly (e.g. `pfb_unbound.dataDB['evil.com'] = {...}`); if left as-is,
the Phase 8 removal of those dicts would force rewriting many tests at the worst
time. Routing fixtures through insert helpers now means the swap later changes
only the helper body.

2. Oracle/property tests for the pure matchers.
   Add a test class that, for a corpus of domains, asserts the matcher outputs.
   Build the corpus two ways and test both:
     (a) Hand-written edge cases that encode the contract explicitly:
         - data exact: 'evil.com' matches 'evil.com', NOT 'x.evil.com'.
         - zone wildcard incl self: 'example.com' matches 'example.com' and
           'a.b.example.com'; matched parent string is returned correctly
           (drives b_eval).
         - whitelist: exact; 'www.' strip; suffix walk respecting tld_seg
           (include a case where a short suffix is BELOW tld_seg and must NOT
           match, and one at/above tld_seg that must match).
         - noaaaa: exact match; wildcard-parent match (parent only — assert a
           wildcard entry on 'example.com' matches 'sub.example.com' but the
           SELF case follows current behavior); confirm parent-only semantics.
         - hsts: TLD-in-hsts_tlds -> ('HSTS_TLD'); suffix membership ->
           ('HSTS'); and explicitly assert the step -2 stride by constructing a
           name where a -1 walk WOULD hit an intermediate label but the -2 walk
           does NOT (this locks the quirk).
     (b) Randomized property test: generate random multi-label domains and random
         blocklist entries; compute "expected" using a straightforward reference
         (you may inline a trivial brute-force suffix check in the test as the
         reference), and assert each matcher agrees. Keep the generator seeded
         for reproducibility.
